# Vital Fitness 版本规划

## 当前进度（约 35-40%）

- ✅ 数据库设计（6张表 + 基础数据）
- ✅ 后端模型层（5个 model，含请求/响应结构体）
- ✅ 后端基础设施（config、logger、database、redis、docker-compose）
- ✅ 后端路由定义（全部 API 路由，处理函数为 placeholder）
- ✅ 前端 UI（13个页面，含完整样式和 mock 数据）
- ❌ 后端业务逻辑（handler / service / dao / middleware 全部为空）
- ❌ 前端数据层（api / store / components / utils 全部为空）
- ❌ 前后端联调

---

## V1.0 - 用户认证系统

**后端：**
- JWT 中间件实现
- 用户注册接口（handler → service → dao）
- 用户登录接口
- 用户登出接口

**前端：**
- API 请求工具封装（request.js）
- Vuex store 搭建（用户模块）
- 登录页对接真实接口
- 注册页对接真实接口
- Token 持久化与自动携带

**验收标准：** 小程序端可完成注册、登录、退出，token 持久化生效

---

## V1.1 - 训练记录模块

**后端：**
- Exercise 动作库查询接口
- Workout CRUD 完整实现（创建、列表、详情、更新、删除）

**前端：**
- 训练添加页对接 API
- 训练列表页对接 API（含分类筛选）
- 动作库从后端加载

**验收标准：** 能添加训练记录、查看训练列表、按分类筛选动作

---

## V1.2 - 体重 + 心情记录模块

**后端：**
- WeightRecord CRUD 接口（含 BMI 自动计算）
- MoodRecord CRUD 接口

**前端：**
- 体重添加/列表页对接 API
- 心情添加/列表页对接 API

**验收标准：** 能记录体重和心情，查看历史列表，BMI 自动计算

---

## V1.3 - 饮食记录 + 用户资料模块

**后端：**
- DietRecord CRUD 接口
- 用户资料查看/更新接口

**前端：**
- 饮食添加/列表页对接 API
- 个人中心/资料页对接 API

**验收标准：** 能记录饮食、查看营养统计、编辑个人资料

---

## V1.4 - 首页仪表盘 + 统计模块

**后端：**
- 统计聚合接口（训练次数、体重趋势、心情均值、饮食汇总）
- 首页概览数据接口

**前端：**
- 首页仪表盘对接真实数据
- 统计页图表对接真实数据

**验收标准：** 首页展示真实数据概览，统计页展示趋势图表
